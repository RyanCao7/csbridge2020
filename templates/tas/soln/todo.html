Problem: Charles Bridge

Teaching tips:

  1. Learning goal: For loops and decomposition
  2. Milestone: build a single column.
  3. Write code on the board. Start with pseudocode, turn to real code
  4. Encourage students to work in pairs to program out the milestones.
  5. The hardest part is the fence-post bug when building a col.

Solution: 

/* BuildCharlesBridge
 * Author: CSBridge Prague
 * Implements a program where Karel fills columns with beepers in order to
 * repair the Charles Bridge. Columns are 4 steps apart and each have a
 * height of 5.
 */

import stanford.karel.*;

public class BuildCharlesBridge extends SuperKarel {
  
  public void run() {
        for (int i = 0; i < 3; i++) {
            buildColumn();
            moveToNextColumn();
        }
        // need to build the last column without moving further east
        buildColumn();
  }

    /* Method: buildColumn
     * -------------------
     * Fills a column (of height 5) with beepers.
     * Precondition: Karel is facing east at the bottom of an empty column.
     * Postcondition: Karel is facing east at the bottom of the column, which
     * is now filled with beepers. 
     */
    private void buildColumn() {
        turnLeft();
        placeBeepersUp();
        comeBackDown();
        turnLeft();
    }

    /* Method: placeBeepersUp
     * ----------------------
     * Puts 5 beepers vertically in a column (of height 5).
     * Precondition: Karel is facing north at the bottom of the column.
     * Postcondition: Karel is facing north at the top of the column, which
     * is now filled with beepers.
     */
    private void placeBeepersUp() {
        for (int i = 0; i < 4; i++) {
            putBeeper();
            move();
        }
        // need to put the last beeper without moving further north
        putBeeper();
    }

    /* Method: comeBackDown
     * --------------------
     * Returns Karel to the bottom of a column of height 5.
     * Precondition: Karel is facing north at the top of the column.
     * Postcondition: Karel is facing south at the bottom of the column.
     */
    private void comeBackDown() {
        turnLeft();
        turnLeft();
        for (int i = 0; i < 4; i++) {
            move();
        }
    }

    /* Method: moveToNextColumn
     * ------------------------
     * Moves Karel from the bottom of one column to the bottom of the next one,
     * which is 4 spaces to the east.
     * Precondition: Karel is facing east at the bottom of one column.
     * Postcondition: Karel is facing east at the bottom of the next column.
     */
    private void moveToNextColumn() {
        for (int i = 0; i < 4; i++) {
            move();
        }
    }
}

Lab

Problem: Mountain Karel

Teaching tips:

  1. Learning goal: while loops and decomposition
  2. Milestone: Ascend the mountain
  3. Encourage students to think about what high level commands might be useful for Karel to know to solve the problem. Then implement those using lower-level Karel commands.
  4. The hardest part is looping to ascend and descend the mountain, knowing the condition to use and what step to take each time.

Solution: 

import stanford.karel.*;

/* This program has Karel ascend a mountain,
 * place a beeper at the top, and then descend
 * a mountain.  The mountain consists of 1 or more
 * 1-level steps.
 */
public class Mountain extends SuperKarel {
  
  public void run() {
    while (frontIsBlocked()) {
        stepUpMountain();
    }
    putBeeper();
    while (frontIsClear()) {
        stepDownMountain();
    }
  }
  
  /* Karel makes one step up a level of the mountain.
   * At the start of the method, Karel is assumed to be
   * facing east towards a wall of the mountain with room
   * above to go to the next level of the mountain.  At the
   * end of the method, Karel is facing east towards a wall
   * of the mountain one level above where Karel started.
   */
  private void stepUpMountain() {
      turnLeft();
      move();
      turnRight();
      move();
  }
  
  /* Karel makes one step down a level of the mountain.
     * At the start of the method, Karel is assumed to be
     * facing east towards the direction where the mountain descends,
     * with room below to go to the previous level of the mountain.
     * At the end of the method, Karel is facing east towards the direction
     * where the mountain descends, one level below where Karel started.
     */
  private void stepDownMountain() {
      move();
      turnRight();
      move();
      turnLeft();
  }
}

Problem: Random Painter

Teaching tips:

  1. Learning goal: if statements, loops and decomposition
  2. Milestone: Paint one row
  3. Encourage students to think about what high level commands might be useful for Karel to know to solve the problem. Then implement those using lower-level Karel commands.
  4. The hardest part is looping between rows, and approaching a larger program.

Solution: 

import stanford.karel.*;

/* Karel paints the entire world randomly, with each
 * corner being randomly either green or blue.  This
 * program works for a world of any size.
 */
public class RandomPainter extends SuperKarel {
  
  public void run() {
      /* Every world has at least 1 row.  Paint this,
       * then paint any additional rows.
       */
    paintRow();
    while (frontIsClear()) {
        moveToNextRow();
        paintRow();
    }
  }

  /* Karel paints a single row in the world, with each
   * square being painted randomly blue or green.
   * At the start of the method, Karel is assumed to be
   * at the leftmost corner of the row facing East.
   * After the method finishes, Karel will be at the
   * leftmost corner of the row facing North, and the
   * row will be randomly colored.
   */
  private void paintRow() {
      paintCornerRandomly();
      while (frontIsClear()) {
          move();
          paintCornerRandomly();
      }
      
      goBackToStart();
      turnRight();
  }
  
  /* Karel randomly paints the corner she is on to be
   * either green or blue.
   */
  private void paintCornerRandomly() {
      if (random()) {
          paintCorner(BLUE);
      } else {
          paintCorner(GREEN);
      }
  }
  
  /* Karel returns to the leftmost corner of the
   * current row.  Before the method starts, Karel
   * is assumed to be at the rightmost corner of a row
   * facing East.  After the method finishes, Karel
   * will be at the leftmost corner of the same row 
   * facing West.
   */
  private void goBackToStart() {
      turnLeft();
      turnLeft();
      while (frontIsClear()) {
          move();
      }
  }
  
  /* Karel moves up one row in the world.  Before the
   * method starts, Karel is assumed to be at the leftmost
   * corner of a row facing North.  After the method finishes,
   * Karel will be in the square above the starting corner,
   * facing East.
   */
  private void moveToNextRow() {
      move();
      turnRight();
  }
}

Mountain Karel

Make sure students use for loops

public class MountainKarel extends SuperKarel {
   public void run() {
      while (frontIsBlocked()) {
         stepUp();
      }
      putBeeper();
      move();
      while (rightIsClear()) {
         dropDown();
      }
   }

   private void dropDown() {
      turnRight();
      move();
      turnLeft();
      if (frontIsClear()) {
         move();
      }
   }

   private void stepUp() {
      turnLeft();
      move();
      turnRight();
      move();
   }

   private void stepDown() {
      move();
      turnRight();
      move();
      turnLeft();
   }
}

Random Painter

Make sure students use for loops

public class RandomPainter extends SuperKarel {
   
   public void run() {
      paintRow();
      returnToWest();
      while(leftIsClear()) {
         turnLeft();
         move();
         turnRight();
         paintRow();
         returnToWest();
      }
   }

   private void paintRow() {
      while(frontIsClear()) {
         paintSquare();
         move();
      }
      paintSquare();
   }

   private void paintSquare() {
      if(random()) {
         paintCorner(GREEN);
      } else {
         paintCorner(BLUE);
      }
   }

   private void returnToWest() {
      turnAround();
      while(frontIsClear()) {
         move();
      }
      turnAround();
   }
}
